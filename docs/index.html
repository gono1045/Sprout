タスクを育てる、
管理Webアプリ

Sprout は、タスクに費やした工数を「EXP」として植物に与え、タスクをこなすほど植物が育っていくタスク管理アプリです。Spring Boot + Thymeleaf を基盤に、ゲーミフィケーション要素・状態管理・アーキテクチャ設計など実務に近い技術を組み合わせて開発しました。

Features

主な機能

工数計測タイマー
タスクごとに作業時間を計測。一時停止・再開に対応し、終了後に作業ログとして記録。
EXP / レベルアップシステム
記録した工数をタグの EXP に変換し、植物（種 〜 大樹・全10段階）が成長していく独自のゲーミフィケーション要素。
インライン編集
テーブル上でタイトル・タグ・ステータス・優先度・締切をその場で編集。複数項目の同時編集を防ぐ排他制御を実装。
タスク管理（CRUD）
タスクの作成・編集・削除・複製・完了切り替え。モーダルとテーブル両方から操作可能。
タグ管理・マイガーデン
カラー付きタグでタスクを分類し、タグ別の植物育成状況を一覧表示（横スクロール対応）。
ユーザー認証・アカウント管理
Spring Security によるセキュアなログイン。パスワード変更・リセットにも対応。
アクセス制御
他ユーザーのタスク・タグへのアクセスを制御する AccessControlService を実装。

What I Learned

この開発で学んだこと

  • Spring Security のカスタム認証フローと UserDetailsService 実装
  • JPA と MyBatis の使い分け（基本CRUDはJPA、複雑なJOIN・動的更新はMyBatis）
  • レイヤードアーキテクチャ（Controller → Service Interface + Impl → DAO）
  • 複数のインライン編集UIが競合しないよう、編集中項目をBlur相当で確定保存してから次を開く排他制御の設計
  • jQueryカスタムイベントによるコンポーネント間の疎結合な状態同期
  • position:fixedなUIにおけるスクロール量の扱いに起因するバグの調査・修正
  • @RequestParamの空配列送信時の挙動（jQuery traditional:trueの落とし穴）
  • Enumによる定数の一元管理（SproutStageでLv閾値・ステージ名を集約）
  • Tailwind CSS 4.x を npm ビルドで組み込むフロントエンド構成
  • Docker を使ったコンテナ化とデプロイ準備
